# Multi PocketBase UI - 제품 기능 명세 (MVP)

이 문서는 MVP 기능 요구사항의 단일 기준 문서다.
타입/상태 모델은 `docs/spec-contracts.md`, 시각 규칙은 `docs/ui-design-spec.md`를 따른다.

## 0) 문서 목적

- 목적: 사용자 관점의 기능 동작과 수용 기준을 고정한다.
- 비목적: 함수명/파일명/구현 순서/프레임워크 선택을 지시하지 않는다.

## 1) MVP 범위

### 1.1 포함
- 인스턴스 등록/수정/삭제
- 관리자 로그인(비영속 인증)
- 컬렉션 조회/검색/즐겨찾기
- 다중 패널 테이블 조회(필터/정렬/페이징)
- 컬럼 표시/숨김/순서/고정(Pin Left)
- 상세 보기(레코드 기준)
- Table View 프리셋 저장/적용
- Workspace 프리셋 저장/불러오기
- 패널 동기화(`filterQuery`, `sortSpec`)

### 1.2 제외
- 레코드 생성/수정/삭제
- 스키마 편집
- 서버 공유 저장
- 다중 컬럼 정렬

## 2) 제품 원칙

1. 인증 정보(`token`, `adminUser`)는 디스크에 저장하지 않는다.
2. 앱 재시작 시 모든 인스턴스는 로그인 필요 상태로 시작한다.
3. 401/403은 인증 만료로 처리하고 재로그인을 유도한다.
4. 동기화는 부분 실패를 허용한다(성공 패널 롤백 금지).
5. 하나의 기능 규칙은 하나의 문서에만 원본으로 둔다.

## 3) 화면 구조

앱은 다음 4영역으로 구성한다.
- Top Bar
- Left Explorer
- Center Workspace Canvas
- Right Inspector

정적 프리뷰(`preview/*.html`)는 구현 검증용 축약 화면으로 사용한다.
프리뷰에서 Inspector를 생략할 수 있으나, 제품 요구사항의 Inspector 기능은 유지한다.

## 4) 기능 요구사항

### 4.1 인스턴스 관리

필수 입력:
- `alias`
- `baseUrl`

규칙:
1. `alias`는 공백 불가, 대소문자 무시 유니크
2. `baseUrl`은 `http://` 또는 `https://`로 시작
3. 활성 인스턴스 삭제 시 다음 인스턴스를 자동 선택(없으면 null)
4. 인스턴스 삭제 시 연결된 인증/즐겨찾기/프리셋 참조를 정리

### 4.2 로그인/인증 만료

규칙:
1. 로그인 성공 시 해당 인스턴스 상태를 사용 가능으로 전환
2. 401/403 발생 시 인증 만료 상태로 전환
3. 재로그인 성공 시 직전 실패 요청을 1회 자동 재시도
4. 로그인 실패 시 기존 데이터는 유지하고 원인 메시지를 노출

재시도 범위:
1. 자동 재시도 대상은 직전 실패한 읽기 요청 1건으로 제한한다.
2. MVP 기준 대상 요청은 컬렉션 목록 조회, 테이블 목록 조회다.
3. 자동 재시도 실패 시 추가 자동 재시도는 하지 않는다.

### 4.3 Explorer

필수 요소:
- 인스턴스 선택
- 컬렉션 검색
- 즐겨찾기
- 컬렉션 목록

규칙:
1. 인스턴스 전환 시 컬렉션 목록 재조회
2. 컬렉션 열기 동작(더블클릭/Enter)은 동일 결과를 보장
3. 즐겨찾기는 즉시 저장되고 재시작 후 복원

### 4.4 패널/테이블

패널 기본값:
- `filterQuery = ""`
- `sortSpec = ""`
- `page = 1`
- `pageSize = 50`

규칙:
1. 컬렉션을 열면 현재 활성 그룹에 새 탭으로 추가한다.
2. 그룹이 없는 초기 상태에서 첫 탭을 열면 루트 그룹을 생성한다.
3. 탭 닫기 후 그룹이 비면 해당 그룹을 레이아웃에서 제거한다.
4. 그룹 제거 후 단일 형제만 남은 split은 자동 정리해 트리를 축약한다.
5. 필터는 Enter로 적용, `Clear`로 초기화
6. 필터 오류 시 기존 정상 데이터 유지
7. 정렬은 `none -> asc -> desc -> none`
8. `sortSpec` 직렬화는 `asc=field`, `desc=-field`, `none=""`를 사용한다.
9. 페이지 인덱스는 1부터 시작한다.
10. 페이징은 경계 페이지에서 비활성 처리한다.
11. 최소 1개 컬럼은 항상 표시한다.

### 4.5 Inspector

탭:
- `Columns`
- `View`
- `Details`

규칙:
1. 활성 패널 변경 시 Inspector 컨텍스트 즉시 동기화
2. `View`는 현재 패널의 조회 설정(`filterQuery`, `sortSpec`, `pageSize`)을 기준으로 프리셋 작업을 수행한다.
3. `View`에서 적용한 프리셋은 동일 `(instanceId, collectionName)` 패널에만 적용한다.
4. `Details`는 선택 레코드 기반으로 표시한다.
5. 선택이 없으면 빈 상태 메시지를 제공한다.

### 4.6 프리셋

Table View 프리셋 저장 대상:
- 컬럼 설정
- 필터/정렬
- 페이지 크기

Workspace 프리셋 저장 대상:
- 레이아웃 트리
- 그룹/탭 구성
- 패널 상태
- 동기화 설정

### 4.7 동기화

규칙:
1. 동일 `syncGroupId` 패널에만 전파
2. 전파 필드는 `filterQuery`, `sortSpec`
3. 대상 패널 실패는 개별 실패로 처리
4. 최신 적용 이벤트를 최종 상태로 인정

## 5) 저장 정책

영속 저장:
- 인스턴스 설정
- 즐겨찾기
- Table View 프리셋
- Workspace 프리셋

메모리 저장:
- 인증 세션
- 인스턴스 런타임 상태
- 일시 에러 컨텍스트

키 네이밍 정책:
- `pbmulti.instances.v1`
- `pbmulti.tableViewPresets.v1`
- `pbmulti.workspacePresets.v1`
- `pbmulti.favorites.v1`

버전 정책:
1. 키 suffix(`v1`)가 다르면 호환되지 않는 데이터로 간주한다.
2. 비호환 버전 데이터는 로드하지 않고 기본값으로 초기화한다.

상세 타입/필드는 `docs/spec-contracts.md`를 원본으로 사용한다.

## 6) 에러 경험 규칙

에러 분류:
- 네트워크 오류
- 인증 만료
- 필터 입력 오류
- 알 수 없는 오류

노출 원칙:
1. 패널 컨텍스트 오류는 패널 내부에서 우선 노출
2. 전역 장애성 오류는 토스트/배너로 노출
3. 사용자 메시지는 원인과 다음 행동(재시도/재로그인)을 함께 제공

## 7) 수용 기준 (MVP)

1. 앱 재시작 후 인증이 복원되지 않고 재로그인을 요구한다.
2. 잘못된 필터 적용 시 기존 테이블 데이터가 보존된다.
3. 동기화 시 한 패널 실패가 다른 패널 성공 결과를 롤백하지 않는다.
4. Workspace 저장/불러오기 후 레이아웃과 패널 상태가 복원된다.
5. 인스턴스 삭제 시 관련 참조 데이터가 함께 정리된다.
6. 탭 닫기로 비어진 그룹이 레이아웃에서 자동 정리된다.
7. 정렬 토글 결과가 `sortSpec` 직렬화 규칙과 일치한다.
8. Inspector `View`에서 저장/적용한 프리셋이 동일 컬렉션 패널에만 반영된다.

## 8) 변경 관리

- 기능 정책 변경은 먼저 이 문서에서 합의한다.
- 타입/상태 반영은 `docs/spec-contracts.md`에서 수행한다.
- 시각/상호작용 반영은 `docs/ui-design-spec.md`에서 수행한다.
