# 기획자 점검 체크리스트 (개발 전달 전)

이 문서는 기획 문서를 개발자에게 전달하기 전에 누락/모호성/중복을 점검하는 표준 절차다.

## 1) 절차

1. 점검 범위 확정: `cli-core-dev-spec`, `ui-mode-dev-spec`, `ui-spec`, `spec-contracts`, `ui-design-spec`, `deployment-cli-brew-spec`, `docs-index`
2. 1차 점검: 아래 체크리스트 항목을 `완료/보완필요`로 판정
3. 보완 수행: `보완필요` 항목만 문서 수정
4. 재점검: 동일 항목을 다시 판정해 전부 `완료`인지 확인
5. 전달 판단: 모든 항목 `완료`일 때만 개발 착수 승인
